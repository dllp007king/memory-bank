# Claude Status Line 设计文档

日期: 2026-03-13

## 概述

为 Claude Code 创建自定义状态栏，显示 Git 状态、上下文窗口使用情况、会话成本和工作目录信息。

## 需求

- 显示信息：Git 状态、上下文窗口使用、会话成本/耗时、工作目录
- 布局：两行混合
  - 第一行：Git 状态 + 上下文窗口使用
  - 第二行：会话成本 + 工作目录
- 实现：Python 脚本

## 架构

### 文件结构

```
~/.claude/
├── statusline.py          # 主脚本
└── settings.json          # 配置文件
```

### 配置

```json
{
  "statusLine": {
    "type": "command",
    "command": "python3 ~/.claude/statusline.py",
    "padding": 1
  }
}
```

## 脚本结构

### 函数列表

| 函数 | 功能 |
|------|------|
| `read_session_data()` | 从 stdin 读取 JSON 数据 |
| `get_git_status()` | 获取 git 分支和文件状态（带缓存） |
| `format_context_bar()` | 格式化上下文进度条 |
| `format_cost()` | 格式化成本显示 |
| `format_directory()` | 格式化目录显示 |
| `colorize()` | ANSI 颜色辅助函数 |

### 主流程

```
读取 JSON → 获取各信息模块 → 组合成两行输出 → 打印到 stdout
```

## 输出格式

### 第一行：Git 状态 + 上下文窗口

```
[main] ✓2 ~1 | [Opus] [▓▓░░░░░░░] 8%
```

| 元素 | 颜色 | 说明 |
|------|------|------|
| `[main]` | 蓝色 | git 分支名 |
| `✓2` | 绿色 | 已暂存文件数 |
| `~1` | 黄色 | 已修改未暂存文件数 |
| `|` | 灰色 | 分隔符 |
| `[Opus]` | 白色 | 模型名 |
| `[▓▓░░░░░░░]` | 根据百分比 | 进度条（10字符） |
| `8%` | 根据阈值 | 使用百分比（<70%绿，70-89%黄，≥90%红） |

### 第二行：会话成本 + 工作目录

```
$0.0123 (23s) | ~/memory-bank
```

| 元素 | 颜色 | 说明 |
|------|------|------|
| `$0.0123` | 绿色 | 累计成本 |
| `(23s)` | 灰色 | API 耗时 |
| `|` | 灰色 | 分隔符 |
| `~/memory-bank` | 白色 | 工作目录（相对主目录） |

## 数据流与缓存

### 数据流

```
Claude Code → stdin (JSON) → Python脚本解析 → 获取Git信息 → 组装输出 → stdout → 状态栏显示
```

### Git 缓存策略

- 缓存文件：`/tmp/claude-statusline-git-cache.json`
- 缓存内容：`{"timestamp": <unix时间戳>, "branch": "main", "staged": 2, "modified": 1}`
- 刷新间隔：5 秒
- 缓存逻辑：
  1. 检查缓存文件是否存在
  2. 如果存在且时间 < 5秒前，使用缓存数据
  3. 否则执行 `git status` 并更新缓存

## 错误处理

| 错误类型 | 处理方式 |
|----------|----------|
| JSON 解析失败 | 输出 `-- \| --`，不崩溃 |
| 字段缺失 (null/不存在) | 使用默认值 |
| Git 命令失败 | 显示 `[git:error]` |
| 缓存文件操作失败 | 降级为直接执行 git 命令 |
| 目录路径处理失败 | 使用原始路径 |

## 测试策略

1. **手动测试**：使用 echo 模拟 JSON 输入
2. **集成测试**：配置后启动 Claude Code 验证
3. **边界测试**：非 git 仓库、百分比 0-100%、成本为 0
4. **缓存验证**：观察 5 秒缓存效果
5. **错误处理测试**：无效 JSON、缺少字段、git 失败
